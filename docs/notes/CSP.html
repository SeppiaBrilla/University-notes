CSP

Note: this is an extension on the [constraints](Fundamentals of Artificial Intelligence and Knowledge Representation/Module 1/Costraints/Constraint satisfaction problem) notes taken for module one of FAIKR course. It is more in depth but cover more or less the same things.

A Costraint satisfaction problem, also called CSP, is a triple <X,D,C><X,D,C><X,D,C> where X is the set of variable to fill, D is the variable domain, C is a set of constraints. A solution to a CSP problem is a variable assignement with domain values that respect the constraints. A CSP problem could also have additional optimization criterion like maximise or minimize some parameters (e.g. costs, profit, ...), so we can extend our CSP definition with a new value f⇒<X,D,C,f>f \Rightarrow <X,D,C,f>f⇒<X,D,C,f> where f is the optimization criterion (a function to minimize / maximise).

Place n queens on an n×nn \times nn×n board. We can model the CSP as follows:

  • Variables, X: one for each row [X1,…,Xn][X_1, \dots, X_n][X1​,…,Xn​], this way we can ensure they don't attach each other row-wise
  • Domain values, D: [1,…,n][1, \dots, n][1,…,n] a value for each column, in this case if we assign values j at variable XiX_iXi​ we will have a queen on row i at column j.
