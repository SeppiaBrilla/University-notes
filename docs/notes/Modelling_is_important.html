Modelling is important

The domain space is exponential in size because, in order to get the full search space, we need to compute the Cartesian product of every variable. Because of that, we need to define our model such that we can prune wrong solution as fast as possible.

There are some tools we can use to improve our models like:

  • Auxiliary variables
  • Implied constraints

An example, the Golomb ruler:

We want to place m marks on a ruler such that the distance between each pair of marks is different and the length of the ruler is minimum. A naive approach:

  • Variables: [X1,X2,…,Xm][X_1, X_2, \dots, X_m][X1​,X2​,…,Xm​], marks on the ruler
  • Domain: {0,1,2,…,2(m−1)}\{0, 1, 2, \dots, 2^{(m - 1)}\}{0,1,2,…,2(m−1)}
  • Constraints: ∀i1,j1,i2,j2,i1≠i2orj1≠j2,∣Xi1−Xj1∣≠∣Xi2−Xj2∣\forall i_1, j_1, i_2, j_2, i_1 \neq i_2 or j_1 \neq j_2, |X_{i_1} - X_{j_1}| \neq |X_{i_2} - X_{j_2}|∀i1​,j1​,i2​,j2​,i1​=i2​orj1​=j2​,∣Xi1​​−Xj1​​∣=∣Xi2​​−Xj2​​∣
  • Objective: minimize(max([X1,…,Xm][X_1, \dots, X_m][X1​,…,Xm​])) With this model the problem is O(m4)O(m^4)O(m4).

A better approach: Introduce a new set of variables, Di,jD_{i,j}Di,j​, representing the distance between the ithi^{\text{th}}ith and the jthj^{\text{th}}jth marks. To express the value of these variables, we must introduce a new set of constraints:

  • ∀i<j,Di,j=∣Xi−Xj∣\forall i< j, D_{i,j} = |X_i - X_j|∀i<j,Di,j​=∣Xi​−Xj​∣
  • alldifferent([D1,2,D1,3,…,D(m−1),m])\text{alldifferent}([D_{1,2}, D_{1,3}, \dots, D_{(m - 1),m}])alldifferent([D1,2​,D1,3​,…,D(m−1),m​]) The "alldifferent" constraint is an implied one while the Ds variables are auxiliary. This new model reduces the computational cost to O(m2)O(m^2)O(m2) which is waaay better.
